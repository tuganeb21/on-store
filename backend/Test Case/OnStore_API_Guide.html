OnStore API
Complete REST API reference for the OnStore e-commerce platform — buyers, sellers, products, orders, chat & analytics.
Base URL http://localhost:5000/api
🔑
All protected routes require a Bearer token in the Authorization header.
Obtain a token via POST /auth/login or POST /auth/register.
🔐 Auth
User registration, login, profile access and password management.
POST /auth/register
Public
Register buyer or seller ▼
Request Body
{
  "username":   "john_doe",
  "email":      "john@example.com",
  "password":   "secret123",
  "role":       "buyer",          // "buyer" | "seller"
  "full_name":  "John Doe",       // buyer only
  "store_name": "My Store",       // seller only (required)
  "phone":      "+250700000001"   // optional
}
Response 201
{
  "success": true,
  "token":   "eyJhbGciOiJIUzI1NiIs...",
  "user":   { id, username, email, role, created_at }
}
POST /auth/login
Public
Login and get JWT token ▼
Request Body
{
  "email":    "john@example.com",
  "password": "secret123"
}
Response 200
{
  "success": true,
  "token":   "eyJhbGciOiJIUzI1NiIs...",
  "user":    { id, username, email, role, avatar_url },
  "profile": { id, store_name }  // or { id, full_name } for buyer
}
GET /auth/me
Auth

Response 200
{

    "id": 1, "username": "john_doe", "email": "john@example.com",
    "role": "buyer", "avatar_url": null, "is_active": true
  }
}

Auth
Change password ▼
Request Body

}
📦 Products
Browse, filter, search, create, update and delete products.
GET /products
Public
List products ▼
Query Parameters
page=1 limit=20 search=keyword category=1 seller=2 min_price=10 max_price=500 sort=price order=ASC
sort options: created_at, price, avg_rating, total_sales, name
Response 200
{
  "data": [{ id, name, slug, price, stock, avg_rating, primary_image, store_name }],
  "pagination": { "page": 1, "limit": 20, "total": 84, "pages": 5 }
}
GET /products/:slug
Public
Product detail ▼
Response 200
{
  "data": {
    id, name, slug, description, price, stock, avg_rating, status,
    "seller": { store_name, seller_profile_id },
    "images": [{ image_url, is_primary, sort_order }],
    "reviews": [{ rating, title, body, username }],
    "priceHistory": [{ old_price, new_price, changed_at }]
  }
}
POST /products
Seller
Create product ▼
Request Body — multipart/form-data
FieldTypeRequiredNotes
namestringrequiredSlug auto-generated
pricedecimalrequirede.g. 49.99
stockintoptionalDefault 0
category_idintoptional
descriptiontextoptional
imagesfile[]optionalMax 8 images, JPEG/PNG/WebP, 5MB each
PUT /products/:id
Seller
Update own product ▼
Same fields as create. Only provided fields are updated. Price changes are automatically logged to price_history.
DELETE /products/:id
Seller
Delete own product ▼
Only the seller who owns the product can delete it. Returns 404 if not found or not owned.
🛒 Cart
Manage the buyer's shopping cart before checkout.
GET/cart
Buyer
Get cart with totals▼
Response 200
{ "data": [{ id, quantity, product_id, name, price, stock, image, store_name }], "total": 149.97, "count": 3 }
POST/cart
Buyer
Add item to cart▼
Request Body
{ "product_id": 5, "quantity": 2 }
Adding an existing item increments the quantity.
PATCH/cart/:id
Buyer
Update quantity▼
{ "quantity": 3 }
Set quantity to 0 to auto-remove the item.
DELETE/cart/:id
Buyer
Remove single item▼
Removes a specific cart item by cart row ID.
DELETE/cart
Buyer
Clear entire cart▼
Removes all items from the buyer's cart.
📋 Orders
Place orders from cart, track history and manage fulfilment per item.
Order Status Flow
pending→ confirmed→ processing→ delivered / cancelled / refunded
POST/orders
Buyer
Place order from cart▼
Request Body
{
  "address_id":     1,
  "payment_method": "mobile_money",
  "notes":          "Pack carefully"
}
Stock is validated and decremented automatically. Cart is cleared on success. Buyer gets a notification.
GET/orders
Buyer
My order history▼
page=1limit=10status=pending
GET/orders/:id
Buyer
Order detail + items▼
{
  "data": {
    id, total_amount, status, payment_status,
    street, city, country,
    "items": [{ product_name, quantity, unit_price, subtotal, item_status, store_name, image }]
  }
}
PATCH/orders/:id/cancel
Buyer
Cancel order▼
Only pending or confirmed orders can be cancelled. Stock is automatically restored.
GET/orders/seller
Seller
Orders with my items▼
page=1status=pending
PATCH/orders/:orderId/items/:itemId/status
Seller
Update item status▼
{ "status": "confirmed" }  // confirmed | delivered | cancelled
Buyer is notified automatically via notification system.
⭐ Reviews
Buyers can review products they have received. One review per product per order.
POST/reviews
Buyer
Submit review▼
{
  "product_id": 5,
  "order_id":   12,
  "rating":     5,     // 1–5
  "title":      "Great quality!",
  "body":       "Really happy with this purchase."
}
Requires item_status = delivered. Product avg_rating is recalculated automatically.
GET/reviews/product/:productId
Public
Product reviews + stats▼
{
  "data":  [{ rating, title, body, username, avatar_url, created_at }],
  "stats": { "total": 42, "avg": "4.70" }
}

⚡ Socket.io — Real-time
Live chat between buyers and sellers, plus instant notifications. Connect with your JWT token.
Connect
const socket = io('http://localhost:5000', {
  auth: { token: 'your_jwt_token_here' }
});
⬆️ Client → Server
chat:open
Create or get room
{ seller_id }
chat:join
Join a room
{ room_id }
chat:send
Send message
{ room_id, body }
chat:typing
Typing indicator
{ room_id, typing: true }
⬇️ Server → Client
chat:room
Room ready
{ room_id }
chat:history
Last 50 messages
[{ id, body, username, sent_at }]
chat:message
Incoming message
{ body, username, sent_at }
notification:new
Real-time notification
{ type, title, body }
Full Chat Flow Example
// 1. Buyer opens a chat with a seller
socket.emit('chat:open', { seller_id: 3 });
socket.on('chat:room', ({ room_id }) => {
  // 2. Join the room to load history
  socket.emit('chat:join', { room_id });
});

// 3. Listen for history
socket.on('chat:history', (messages) => renderMessages(messages));

// 4. Send a message
socket.emit('chat:send', { room_id: 7, body: 'Is this still available?' });

// 5. Receive messages in real-time
socket.on('chat:message', (msg) => appendMessage(msg));
